最长回文

马拉车算法简直是太有意思了，将一个暴力时间复杂度为O(n^3)的问题优化成了一个时间复杂度为O(n)的问题。不得不说，很牛逼，值得记录一次。

问题描述

求给定字符串中的最长回文

什么是回文？类似”abba”、”acdca”这样正序和倒序完全相同的字符串就可以称为回文

暴力算法

一般碰到回文的问题，可以通过双指针来判断，暴力算法解决
枚举所有子字符串，判断各个子字符串是否是回文，记录最长的那个。
这种算法最简单，但是时间复杂度也是最高的O(n^3)，并且不能通过Leetcode的测试用例

动态规划

万能的动态规划解法，只要是涉及到动态规划的问题，就算是比较难的问题了吧。
暴力算法子串的首位遍历，加上每一个子串是否为回文的遍历，组成了三层遍历，也将时间复杂度贡献到了O(n^3)。如果我们保存每一个子串是否为回文的状态，根据上一个状态来推测当前的状态，这样是否可行呢？当然是可行的！这是一种用空间换时间的方法，在暴力解法的基础上减少了一次遍历，只需要遍历首位子字符串就可以了，成功将时间复杂度降到O(n^3)。

由于我们想用上一次的状态来判定当前的状态，所以我们定义状态数组dp[i][j]，表示以i开头，j结尾的子字符串是否为回文串。
定义好了状态数组，我们再来定义状态转移方程：

  • dp[i][j] = dp[i + 1][j - 1] && s[i] == s[j]

如果去掉首位两个字符串后的子字符串是回文字符串，且首位字符相等，那当前的子串也一定是回文串。

确定好了状态转移方程，我们再来确定边界条件：
1. 如果i == j，即只有一个字符的时候，这个字符当然是自己的回文
2. 如果i == j - 1 && s[i] == s[j]，即只有两个字符，且两个字符都相等的时候，这两个字符当然是回文
3. 如果i < j，当前子串是不存在的，因为不存在一个子字符串的开始比结尾大

根据状态转移方程，上一个状态的长度肯定比现在的长度要小2，所以我们把长度作为第一层循环，从0到字符串的长度遍历。
第二层循环以开始位置为起点，以结束位置小于字符串长度为终点执行遍历
由于长度在遍历中是递增的，所以后面找到的回文字符串肯定比前面的长，所以保存最后一次找到的回文字符串就可以了。

但是。。。但是。。。在swift下这种方式还是翻车了，依然不能通过上述的测试用例。虽然时间复杂度已经降到了O(n^2)，空间复杂度升到了O(n^2)，但还是慢了。
不知道是因为字符串的处理慢，还是其它什么因素导致的有问题。尝试过各种修改，但是在当前思路下，还是没有办法通过测试用例，不得不换一种思路。
